# my-library

A personal library of technical learning topics — novelized into chaptered narratives. Each book is a study artifact: I take a subject I'm trying to internalize and rewrite it as a guided journey, chapter by chapter.

## Books

| Book | Subject |
|---|---|
| `claude-architect` | Claude Certified Architect Foundations — orchestration, tools, structured output, evals |
| `claw-code-python` | Claude Code internals through a Python lens |
| `cloud-pavilion` | Cloud + AI deployment, an eight-layer mental model |
| `gemma-shop` | Local Gemma 4 inference + agent framework design |
| `inference-engineering` | ML inference systems — batching, KV cache, multi-LoRA routing |
| `openclaw-odyssey` | Reverse-engineering openclaw, scoped to the agent invocation closed loop |

Each book is a directory containing a `book.json` (title, author, cover, chapter list) and chapter markdown files (`ch00-map.md`, `ch01-…md`, …).

## Why novelize

Technical material drifts out of working memory if I read it like documentation. Rewriting it as a chaptered journey forces me to pick a narrator, choose a route through the topic, and decide what each chapter has to deliver. That structure is what survives.
