# 洞窟国 · CLAUDE.md 的三层阶梯

> 这章要解决什么问题？
> Claude Code 的配置是怎么分层的？`CLAUDE.md` 三层什么意思？Plan Mode 和 CI 集成又是哪门子事？

---

## 一

洞窟国的首都在一座山上——**CLAUDE.md 山脉**。

山脉从上到下有三层阶梯。每一层，都是一层 context 的叠加。

```
最高层：~/.claude/CLAUDE.md         (用户级 — 你所有项目都会读到)
中间层：<repo>/CLAUDE.md            (项目级 — 这个仓库的所有子目录都会读到)
最低层：<repo>/<subdir>/CLAUDE.md   (目录级 — 只有在这个子目录下 agent 才会读到)
```

旅行者一开始以为这是一种"继承"——父级的规则子级继承。读了几页他才发现——**是叠加**。

- 最高层说"我喜欢 TypeScript"
- 中间层说"这个项目用 Python"
- 最低层说"这个子目录用 Rust"

最后 agent 的脑子里是三份同时在场的指令。后面的会**覆盖**前面的具体点（"现在这个 subdir 用 Rust"），但**泛指令会保留**（"我喜欢类型安全"——不冲突）。

旅行者在笔记里写:

> **CLAUDE.md 三层是分级的声明, 不是继承链**.
> 细节以最近的层为准, 风格以全局为底.

---

## 二

山脉中段有一个入口——写着 **Plan Mode**。

旅行者走进去, 里面是一个图书馆, 桌上摆着一本厚厚的开放笔记本——**计划书**。

管理员说:"这个模式下, agent 不直接动手. 它先在这本本子上写一个计划: 我打算做 ABC, 用 XYZ 工具, 预计会碰到 123 风险."

"然后呢?"

"然后**用户读完后说'开始'**, 它才开始执行."

"为什么不直接让它做?"

"因为——"管理员顿了一下, "复杂任务的失败成本是滚雪球. agent 一上来就改 20 个文件, 有 1 个方向错了, 后面 19 个都要回退. Plan 让错误在**纸上**发生, 不在**代码**里发生."

"所以 Plan Mode 不是'慢', 是'把昂贵的错误发生地点从代码挪到纸上'."

旅行者在阶梯上记下这条:

> **复杂度高 → Plan Mode, 复杂度低 → 直接执行.**
> 判断标准: 回退成本.

---

## 三

山脉的另一侧, 他遇到了一群穿着流水线制服的工人——**CI 工**.

他们用的是 `claude -p "<prompt>"` 这个命令. `-p` 是 prompt 模式, 非交互式. agent 跑完就退出, 把结果打印出来.

旁边有另一群工人, 跑的是**独立审查实例**——

```bash
# 主 CI 实例: 跑测试, 写代码
claude -p "fix the failing test" > result1.md

# 独立审查实例: 读上面实例的输出, 评判它做得对不对
claude -p "review this diff, is it correct?" < result1.md
```

"一个 agent 审查自己的结果, 靠谱吗?" 旅行者问.

"不靠谱." 工人说. "自审有偏见——它会**合理化**自己刚做的决定."

"所以要用**独立实例**——新的 context, 不知道前面的推理过程, 只看最终产出."

这让旅行者想起编排国的 coordinator-subagent. 他突然明白——**多实例评审 = 一种 subagent 模式的特例, 专门用来反制'自审偏见'**.

---

## 四

旅行者在洞窟国的最后一夜, 在山脚下的温泉边写了一段总结:

> CLAUDE.md 三层像**一栋房子的灯光**——
> - 主灯（用户级）: 我的整体光线
> - 台灯（项目级）: 这个桌子的局部光线
> - 射灯（目录级）: 这本书的点光
>
> Plan Mode 是**在施工前画图纸**——不是慢, 是把改动的成本前移.
>
> CI + 独立审查是**流水线上的 QA + 独立审计**——同一个人做又 QA 又审计, 审不出来.

---

## 关键概念

- **CLAUDE.md 三层**：用户级 / 项目级 / 目录级。**叠加不是继承**；细节以最近的层为准，风格以全局为底。
- **Plan Mode**：agent 先写计划再执行。适用于**回退成本高**的复杂任务。
- **CI 集成 `-p`**：非交互式 prompt 模式。用**独立实例**做审查，避免自审偏见。

## 我的理解

- **配置分层**的核心目的不是"复用"，是"关注点分离"——我的风格 vs 项目约定 vs 此刻场景
- **Plan Mode** 和**普通执行**的选择不看任务难度，看**错误回退成本**
- **独立审查实例**和 coordinator-subagent 是同一个思想在**质量保证**这个维度上的投影

## 类比

- CLAUDE.md 三层像**穿衣服**: 内衣（user）、外套（project）、围巾（dir）。每一层都能加减，不冲突就叠加。
- Plan Mode 像**施工图**. 盖房子前先画图, 图错了改图很便宜, 墙砌了再拆就惨.
- 独立审查像**法庭上的检察官和辩护律师**——不能让同一个人两个角色都当.

## 遗留问题

- CLAUDE.md 层级冲突怎么解? 我猜是**最具体的层赢**(dir > project > user), 但考试可能会问 edge case.
- Plan Mode 的"执行" phase 能不能中断再回 plan? 没仔细想过.
- 独立审查用什么 prompt 最有效? "review this diff" 还是 "pretend you didn't write this"?
