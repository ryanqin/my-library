# 序章 · 一张地图，五个国

> 这章要解决什么问题？
> 架构师的考试到底考什么？把 5 个 domain 画成一张地图——接下来的六章，就是这张地图上的六段旅行。

---

## 一

他在背包里塞了三样东西：一本考卷、一支笔、一张画了五个国家的地图。

"六天。" 他跟自己说。"五个国家走一圈，第六天回来考试。"

地图上写着:

| 国家 | 考试里占多少 | 首都 |
|------|-----------|------|
| 编排国 | 27% | stop_reason 城 |
| 工具国 | 18% | MCP 镇 |
| 洞窟国 | 20% | CLAUDE.md 山脉 |
| 结构国 | 20% | Few-shot 街 |
| 镜厅国 | 15% | 上下文回廊 |

五个国家旗帜不同，语言互通。都是**同一件事情的五种侧面**——"怎么让一个会说话的 LLM 变成一个能做事的 agent"。

---

## 二

他在地图背面写了一段话——**给自己的出发须知**:

```
旅行者须知 (来自一位旧时代的架构师)

1. 每一国都会有一个推销员, 他会说他的国家最重要.
   别信. 五国互为依托, 没有哪个国单独能产出一个 agent.

2. 每一国都有一张反模式墙.
   墙上贴满当地人做错的事——那些比"正确做法"更值得抄下来.
   记住: 考试考的是"看出别人做错",不是"自己做对".

3. 每一国都有一个陷阱.
   陷阱不是"难",是"看着对,实际错"——
   它们把你训练出的直觉反过来用.
   要走得慢一点.
```

他把地图折好，放进内袋——今晚他要动身。

---

## 三

当晚他睡得不太好。

在梦里，五个国家的国王围着他问同一个问题——
"如果只能选一件事带回去，你选什么？"

他说不出来。

他梦到自己站在五个国家的边境，同时看见五种光——

- 编排国的光是**蓝色**，像 stop_reason 的状态跳动
- 工具国的光是**金色**，像 tool 描述被 LLM 扫过时的瞬间
- 洞窟国的光是**土色**，像 CLAUDE.md 在多层目录里的沉默堆叠
- 结构国的光是**白色**，像 JSON 结构的棱角
- 镜厅国的光是**彩色的碎片**，像丢失的 context 被镜子折射出的错觉

五种光叠在一起，他看见一个 agent 站在中央——像一个在五束灯下的演员。

他醒过来，把这个梦记下来。

"**五国不是选一个**。" 他在笔记开头写下。"**五国是五根柱子，少一根 agent 就塌。**"

---

## 关键概念

- **Agentic orchestration**（编排国）：`stop_reason` 驱动的循环，coordinator-subagent 模式，Hooks vs Prompt 强制。
- **Tools & MCP**（工具国）：Tool 描述决定 LLM 选 tool，`tool_choice` 三种模式，MCP 是工具的跨 agent 合约。
- **Claude Code**（洞窟国）：CLAUDE.md 的用户/项目/目录三层，Plan Mode vs 直接执行，`-p` CI 集成。
- **Structured output**（结构国）：Few-shot 是模糊场景下最强的对齐手段，tool_use 强制 JSON，Message Batches API。
- **Context & reliability**（镜厅国）：Lost-in-the-middle，Escalation 决策，多实例评审 ≠ 自审，Provenance 追溯。

## 我的理解

五个 domain **不是分区**，是 **agent 的 5 个失败模式**的对立面：

- 不编排好 → agent 死循环
- 不会描述 tool → LLM 不用那个 tool
- 不配 CLAUDE.md → agent 不知道你在哪个项目
- 不给结构化模板 → agent 给出一段散文
- 不管 context → agent 回答"很有道理"的错话

**五国 = 五种失败的反面**。

## 类比

像武侠小说的"五岳剑派"。每一派都是一套招数,单练任一派都能杀人,但**五派的真正价值是"互为弱点的补位"**。独孤九剑说破了——"无招胜有招"——其实是**五种招数都吃透后的融合**。

## 遗留问题

- 考试会不会出"跨 domain"的场景题？我怀疑会——agent 的真实 bug 从来不只属于一个 domain。
