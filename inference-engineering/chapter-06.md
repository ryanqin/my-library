# 第六章：变形

## 不只是文字

P.K.把我带到一个多媒体实验室。

四面墙上的屏幕显示着不同的内容：一边在实时转录语音，一边在生成图像，还有一边在分析视频。

"LLM是最著名的生成式AI，但远不是唯一的。"P.K.说，"AI现在能看、能听、能说、能画。每种模态都有自己的推理挑战。"

他递给我一份表格：

| 输入 | 输出 | 类别 |
|-----|-----|-----|
| 文本+图像/视频 | 文本 | 视觉语言模型（VLM） |
| 文本或图像 | 向量 | 嵌入模型 |
| 音频 | 文本 | 语音识别（ASR） |
| 文本 | 音频 | 语音合成（TTS） |
| 文本 | 图像 | 图像生成 |
| 文本 | 视频 | 视频生成 |

"好消息是，只有两种基本范式：**自回归token生成**和**迭代去噪**。大多数模态都能归到其中之一。"

---

## 视觉语言模型（VLM）

### 架构

VLM = LLM + 视觉编码器

视觉编码器把图像/视频转换成token，LLM处理这些token和文本token。

视觉编码器很小（通常几十亿参数），LLM很大（几百亿到上千亿参数）。比如Mistral Large 3：视觉编码器20亿参数，LLM 6730亿参数。

### 关键挑战：长上下文

一张高清图像 ≈ 1000个视觉token。

这意味着：
- **预填充更长**：图像token需要被patch、嵌入、tokenize
- **KV缓存更大**：更长的上下文 = 更多的KV存储

### 优化策略

所有LLM技术都适用，但更重要：

| 技术 | 对VLM的作用 |
|-----|------------|
| KV缓存量化 | 减少长上下文的内存开销 |
| 前缀缓存 | 多轮对话中复用图像的KV |
| 分离 | 长输入的预填充独立扩缩 |
| 张量并行 | 更多显存支持大模型+长上下文 |

### 视频处理

一秒24帧。每帧是一张图像。

4秒视频 = 96帧 ≈ 96,000个token（理论上）

实际中必须**降采样**：降低分辨率和帧率。

即便如此，视频理解模型只能处理很短的片段。前缀缓存和KV卸载对视频尤其重要。

## 嵌入模型

### 什么是嵌入

把可变长度的文本（或其他数据）转换成固定长度的向量。向量编码了语义信息。

```
"今天天气不错" → [0.12, -0.45, 0.78, ..., 0.33]  (比如768维向量)
```

用途：RAG、搜索、推荐系统、智能体记忆。

### 两种流量模式

1. **高吞吐量批处理**：索引百万文档、更新产品目录
2. **低延迟查询**：用户搜索、实时检索

如果两种都需要且流量够大，**分开部署**。

### 架构

两种主流：

| 类型 | 参数量 | 特点 |
|-----|-------|-----|
| BERT风格 | <1B | 编码器专用，简单任务用 |
| LLM改造 | ≤8B | 能力更强，现代主流 |

### 推理优化

- **TensorRT-LLM**：XQA内核优化注意力，内核融合减少内存访问
- **FP8量化**：小模型质量敏感，但FP8通常可接受
- **高批处理**：嵌入模型支持很高的批处理量
- **水平扩展**：每个GPU作为独立副本

**质量检查**：量化后，用余弦相似度比较原始和量化后的输出向量。目标：≥99%相似度。

## 语音识别（ASR）

### Whisper

最流行的开源ASR模型。最大版本只有15.5亿参数，在GPU的一小部分上就能跑得很快。

架构：编码器-解码器

```
音频波形 → 编码器 → 音频特征 → 解码器 → 文本token
```

解码器是瓶颈——它是一个自回归Transformer，和LLM类似。用TensorRT-LLM优化效果最好。

### 两种场景

**实时转录**

用户在说话，系统需要实时显示文字。

关键指标：**流式延迟**——从用户说完到文字出现。

优化策略：
- 音频分块处理（比如每2秒一块）
- 用推测解码降低延迟
- 并行处理多个音频块

**离线转录**

处理长音频文件（播客、会议）。

关键指标：**端到端时间**——整个文件处理完的总时间。

优化策略：
- 把长文件切成多块，并行处理
- 高批处理量
- 优先吞吐量而非延迟

### 说话人分离

识别"谁在说话"。

通常需要额外的模型，和ASR组成流水线。

---

## 语音合成（TTS）

### 让AI说话

TTS把文本变成自然的语音。用于语音助手、有声书、AI主播。

### 架构演变

早期：拼接合成（听起来很机器）  
现在：神经网络端到端生成（几乎和真人一样）

现代TTS模型也是Transformer架构，输出的是音频token。

### 关键指标

不是TTFT，而是**首句时间**——因为一个音频token没意义，需要整句话。

### 流式TTS

语音对话需要实时合成。

挑战：生成速度必须超过播放速度。

```
播放速度：1秒音频 / 秒
生成速度：需要 > 1秒音频 / 秒（最好是2-3倍）
```

如果生成速度跟不上，就会卡顿。

优化：
- 小模型（几十亿参数）
- 量化
- 流式输出（边生成边播放）

### 语音到语音

新兴的模型直接处理语音输入、输出语音，跳过文本中间步骤。

优点：更自然、更快  
挑战：更大的模型、更复杂的推理

## 图像生成

### 不同的范式

图像生成不是自回归的。它用**扩散**：从纯噪声开始，逐步去噪，显现图像。

```
步骤0: [噪声] 
步骤1: [噪声 - 一点噪声]
步骤2: [噪声 - 更多噪声]
...
步骤N: [清晰图像]
```

步数越多，质量越高，速度越慢。

### 流水线

图像生成不是单一模型，是多个模型的流水线：

1. **文本编码器**：把提示词变成嵌入向量
2. **扩散模型**（UNet或DiT）：核心，迭代去噪
3. **VAE解码器**：把潜在空间转成像素

### 优化关键

**计算是瓶颈**（不是内存）。

策略：
- **注意力优化**：FlashAttention、xFormers
- **量化**：扩散模型对量化敏感，但FP8通常可行
- **内核融合**：减少扩散步骤之间的内存访问
- **减少步数**：蒸馏模型用更少步数达到相似质量

### "一个奇怪的技巧"

在扩散过程中，早期步骤决定大结构，后期步骤决定细节。

可以在后期步骤**降低分辨率**计算，然后上采样。节省计算但几乎不损失质量。

---

## 视频生成

### 最前沿

视频生成是最新、最有挑战性的模态。

一个视频 = 很多帧 × 每帧很多像素 = 巨大的计算量

### 架构

主流是**扩散Transformer（DiT）**——结合扩散和Transformer的优点。

注意力需要跨越空间（帧内像素）和时间（帧之间）。

### 优化策略

| 技术 | 作用 |
|-----|-----|
| 注意力优化 | 空间和时间注意力分开处理 |
| 量化 | FP8，但视频模型更敏感 |
| 上下文并行 | 把长序列分到多GPU |
| 分块处理 | 视频分成重叠的块并行处理 |

### 特有挑战

- **VRAM爆炸**：长视频的中间状态很大
- **一致性**：保持帧之间的连贯
- **音频同步**：如果有音频，还需要对齐

视频生成仍在快速发展。推理优化技术也在不断更新。

## 模态之间的共性

P.K.在白板上画了一个总结图：

```
                     自回归           迭代去噪
                        │                │
            ┌───────────┼────────────┐   │
            │           │            │   │
          VLM         ASR          TTS   │
          嵌入                           │
                                         │
                                   ┌─────┼─────┐
                                   │     │     │
                                 图像  视频  3D
```

"自回归模型（VLM、ASR、TTS、嵌入）的优化技术基本相通。"

"扩散模型（图像、视频）有自己的一套技巧。"

"但无论哪种，核心问题都是一样的：**找到瓶颈，针对性优化**。"

## 档案室

**本章核心概念：**

1. **两种范式** = 自回归token生成（VLM、ASR、TTS、嵌入）+ 迭代去噪（图像、视频）。前者的LLM优化技术大多适用，后者有独立的优化路径。

2. **VLM关键** = 长上下文处理。图像/视频变成大量token，KV缓存量化、前缀缓存、分离特别重要。

3. **图像/视频关键** = 计算是瓶颈。注意力优化、减少扩散步数、内核融合是主要优化方向。
