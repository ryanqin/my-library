## 神经网络的基本单元

第一份文件标题是《解剖指南》。

**神经元（Node）**

神经网络的最小单元。它做一件简单的事：
1. 接收输入
2. 乘以权重
3. 加上偏置
4. 返回结果

数学上就是：`y = Wx + b`

**层（Layer）**

一组神经元组成一层。同一层的神经元互相独立，各算各的。但层与层之间是连接的——上一层的输出是下一层的输入。

LLM的神经网络有几十到几百层。

**三种层**：
- **输入层**：接收原始输入
- **隐藏层**：中间所有的层，逐步转换输入
- **输出层**：生成最终预测

---

## 矩阵乘法：最核心的运算

文件的下一节让我意识到为什么GPU对AI如此重要。

神经网络中最本质的运算是**矩阵乘法（matmul）**。

一个向量（一列数字）通过一个矩阵（一个数字网格）相乘，得到一个新向量。

```
y = Wx + b

其中：
- x 是输入向量
- W 是权重矩阵
- b 是偏置向量
- y 是输出向量
```

权重矩阵的值是在训练时确定的。推理时，我们只是不断地做矩阵乘法。

---

## 激活函数：打破线性

但有一个问题。

矩阵乘法是可组合的。如果一个向量乘以矩阵A，再乘以矩阵B，等于直接乘以矩阵A×B的乘积。

这意味着：一系列的线性层会**坍缩成一层**。那多层神经网络就没意义了。

解决方案：在层与层之间插入**激活函数**。

激活函数是非线性的。最简单的是**ReLU**：
- 如果 x > 0，返回 x
- 如果 x ≤ 0，返回 0

就这么简单。但它打破了线性，让多层网络成为可能。

现代模型用更复杂的激活函数——SiLU、Swish、SwiGLU——但原理相同：快速、可微分（支持反向传播）、打破线性。

## Transformer：一切的起点

文件堆里有一篇被圈出重点的论文，标题是：*"Attention Is All You Need"*，2017年。

这篇论文改变了一切。

**Transformer** 是一种带有注意力机制的神经网络，能学习序列中不同部分之间的关系。

Transformer 是生成式AI的基础。不只是LLM——语音、图像、视频生成，都基于Transformer。

两种主要的Transformer模型风格：

1. **自回归token生成**：从一个token序列开始，预测下一个最可能的token。这是LLM的工作方式。

2. **迭代去噪**：从随机噪声开始，逐步精炼到最可能的输出。这是图像生成的工作方式。

## LLM是如何"思考"的

我在文件里发现了一份关于LLM推理机制的详细说明。

**Token：语言的原子单位**

LLM不处理文字，处理的是token——代表文本块的数字。

现代LLM使用**子词分词**：常见词是一个token，不常见的词会被拆成多个token。

比如："understanding" 可能被拆成 "under" + "stand" + "ing"

分词器（tokenizer）是一个简单的映射表，不需要神经网络。每个模型有自己的词汇表，通常超过100,000个token。

**推理的两个阶段**

1. **预填充（Prefill）**：处理输入序列，为每个输入token计算注意力，存入KV缓存
2. **解码（Decode）**：逐token生成输出，每生成一个token就更新KV缓存

每次解码的前向传播产生一个**logits向量**——长度等于词汇表大小，每个位置代表该token是下一个token的概率。

然后通过temperature、top-k、top-p等参数控制如何从这个概率分布中采样。

## 注意力机制：Transformer的灵魂

文件里有一页被反复标记，标题是"Attention"。

**为什么需要注意力？**

考虑这个句子："我决定写一本书，因为我觉得它会很简单，但实际上它很难。"

句子中的"它"指的是什么？人类很容易理解是"写书"这件事。注意力机制让LLM也能理解这种关系。

**注意力方程**

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

三个输入：
- **Q (Query)**：当前token的表示
- **K (Key)**：所有之前token的表示
- **V (Value)**：所有之前token的计算值

**多头注意力**

注意力层有多个"头"，并行运行。每个头可能负责不同类型的关系：主谓一致、指代消解等。

**KV缓存**

注意力需要查看当前token与所有之前token的关系。如果每次都重新计算，时间复杂度是 O(n²)。

解决方案：把之前token的K和V存起来，这就是**KV缓存**。

KV缓存在预填充时构建，在解码时使用和更新，默认存在GPU显存里。它是推理工程中最重要的优化对象之一。

## 混合专家模型（MoE）

文件堆的最后是一份关于"专家"的报告。

传统神经网络是**稠密**的——每个输入都经过所有参数。

**混合专家模型（Mixture of Experts）** 引入了稀疏性。

不是一个巨大的矩阵，而是几百个小矩阵（专家）。每个输入只会激活其中一小部分专家。

例如：Qwen3-235B-A22B 总共有2350亿参数，但每次请求只激活220亿参数。

**路由器**

一个小模型负责决定激活哪些专家。在生成每个token时，路由器在每一层选择要使用的专家。

**为什么重要**

MoE模型在单请求推理时非常高效——只有一小部分参数被使用。

但在批处理时，不同请求激活不同的专家，几乎所有参数都会被用到。

这是推理优化的一个重要考虑因素。

## 图像生成：另一种范式

当我以为理解了一切，文件里出现了图像生成的章节。

图像生成模型不是单一模型，而是**多个模型组成的流水线**：

1. **文本编码器**：把提示词转换成嵌入向量
2. **扩散模型**：核心，通过迭代去噪生成图像
3. **VAE解码器**：把潜在空间转换成像素

**扩散的原理**

从纯噪声开始，每一步预测并移除一点噪声，逐步显现出图像。

步数越多，质量越高，但速度越慢。现代模型通过各种技术减少所需步数。

## 瓶颈在哪里？

我在文件的最后发现了最关键的部分。

GPU有两个主要资源：
- **计算能力**：每秒能做多少次运算
- **内存带宽**：每秒能搬运多少数据

不同的操作受限于不同的瓶颈：

**LLM预填充**：计算受限。大量token需要并行处理，GPU的计算单元是瓶颈。

**LLM解码**：内存受限。一次只生成一个token，需要不断从显存加载模型权重，带宽是瓶颈。

**图像生成**：计算受限。大量的注意力和卷积运算，计算是瓶颈。

理解瓶颈是优化的前提。你不能优化一个不是瓶颈的东西。

## 档案室

**本章核心概念：**

1. **Transformer架构** = 神经网络 + 注意力机制。自回归token生成（LLM）和迭代去噪（图像生成）是两种主要范式。

2. **LLM推理两阶段** = 预填充（构建KV缓存）+ 解码（逐token生成）。预填充是计算受限，解码是内存受限。

3. **注意力与KV缓存** = 注意力让模型理解token间的关系。KV缓存把O(n²)变成O(n)，是推理优化的核心对象。
