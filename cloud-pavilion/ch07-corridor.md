# 第七层 · 图书馆走廊与会议室

> 这章要讲什么?
> RAG 与 Agent 编排. 从"demo 能跑"到"生产能稳"的鸿沟.

---

## 一

门里面, 是一条**图书馆走廊**.

两边是一排排到顶的书架, 书脊整整齐齐. 但每一本书的封面都没有字——只有一串短短的数字.

阿衍拿下一本, 翻开. 里面也没有字, 只有**密密麻麻的小数**.

"Embedding." 老周说, "每一本书都是一段文档, 被切成块, 又被编码成向量. 这个书架, 是一个向量库."

"那怎么查?"

老周挥手, 走廊尽头一扇屏幕亮了. 上面写着:

> 用户问题: 「去年 Q3 印度市场退货率高的原因?」

屏幕上, 一个光点从问题出发, **在书架间飞**. 它飞得很快, 十几秒内扫过几百本书, 最后停在十几本上.

"Top-K 检索. 向量相似度最高的前十几份."

阿衍注意到, 这十几本书被**重新排了一次序**——有几本往前移了, 有几本被淘汰了.

"这是?"

"Rerank. **检索员的第二次筛选**. 因为 embedding 相似不等于真的相关. Rerank 用一个更精细的模型再打一次分. **没有 rerank 的 RAG, 是半成品**."

---

## 二

被选中的十几本书, 被抽出来, 送进走廊尽头的一扇门.

门上写着:

> **会议室 · Agent Orchestration**

阿衍推门进去.

是一间**圆桌会议室**. 坐着三个人.

最上面那个, 胸前挂着名牌 `Planner`. 他手里拿着刚才送来的那十几本书, 在翻.

他对面坐着一个名牌是 `Tool Router` 的人, 桌子上摆着一堆小牌子: `SQL`、`Search`、`Calculator`、`Slack`、`Email`、`Shell`……

还有一个 `Memory`, 在角落做笔记.

---

## 三

阿衍站在门口. 会议开始了.

**Planner**: 「我需要去年 Q3 印度的退货数据. 先查数据库.」
**Tool Router** 扔出 `SQL` 的牌子.
——屏幕跳出 SQL 查询结果.
**Planner**: 「数据有了, 但没有原因. 去搜一下新闻.」
**Tool Router** 扔出 `Search`.
——搜索结果进来.
**Planner**: 「有一条提到物流合作伙伴换了. 交叉验证一下, 问财务 Slack 频道.」
**Tool Router** 扔出 `Slack`.
……

阿衍看着这一幕, 忽然觉得——**这像他以前带过的一个实习生**. 不, 更像**三四个实习生的协作**.

"这就是 Agent." 老周说, "**Planner 决定下一步, Tool Router 选工具, 拿到结果, 继续迭代. Memory 记下来**, 让它下一轮还知道自己干了什么."

---

## 四

会议开了十分钟, Planner 忽然停了下来.

屏幕上跳出一行红字:

> ⚠ max_steps 达到 20. 强制停止.

阿衍吓了一跳. "它不是正在工作吗, 为什么停?"

老周指了指红字下面一行小小的:

> 累计消耗: 42,000 tokens · $0.85

"这一次调用花了将近一块钱. 一个用户问题. 如果没有 max_steps, 它可以一直查下去——**Planner 自我怀疑, 又加一轮; tool 失败, 又重试一轮**. 一天下来, 就是几万块."

阿衍想起下午那本账本.

他明白了: **八楼最大的风险, 不是跑不动, 是跑不停**.

---

## 五

会议室旁边, 还有一个小房间.

门上写着: **Eval**.

阿衍 这 次 没 探头——他 直接 推 门 进 去.

里面 比 他 想 的 安静. 一个 机器 人 坐 在 桌前, 它 不 抬 头, 手指 在 键 盘 上 飞快 地 敲. 房间 角落 有 一台 老式 的 打印机, 一张 一张 慢 慢 吐出 测试 报告——**纸 上 的 字 是 红 的 和 绿 的, 像 圣诞树 的 装饰**.

阿衍 走 到 机器 人 旁边, 看 它 屏幕.

屏幕 是 一个 表格. 第一 列 是 问题, 第二 列 是 标准答案, 第三 列 是 Planner 的 回答, 第四 列 是 分数.

他 弯 下腰, 凑 近 一行 看:

```
Q  : "去年 Q3 印度退货率 高的原因 是什么?"
GT : "物流 合作伙伴 在 7 月 末 切换, 新 伙伴 在 季风 季节 节点 准备 不足"
A  : "物流 合作 切换 + 季风 + 仓储 损坏"
分数: 0.78
```

阿衍 想 了 想, 觉得 0.78 有 道理——**A 比 GT 多 了 一 句"仓储 损坏"**. 这 一句 是 不是 对 的? 数据 里 没说. Planner 是 不是 编 的?

——他 抬 头 问 老周:

"——这个 0.78, 是 怎么 算 出 来 的?"

"semantic similarity + 关键事实 命中 + LLM-as-judge. 三 个 维 度 加权."

"那 如果 Planner 编 了 一 个 ' 看起来 对 但 实际 没 数据 支持' 的 答案——"

"——就 会 像 这条 一 样, **0.78**. 不 是 满分, 但 也 不 是 0 分. 我们 要 看 的 是 这种 0.78, 它 是 **真的 接 近 GT, 还是 编 得 像**.

——这 也 是 为什么 我 们 在 屏幕 后面 又 挂 了 一台 打印机. 每 一 条 < 0.85 的 case **必须 人 工 复核**, 写 在 这 些 红绿 纸 上, 第二天 早会 review."

阿衍 拾 起 一张 刚 打 出来 的 红 字 报告. 上面 一行 用 笔 圈 出 来:

> **此 答案 数据 不存在, Planner 幻觉. 加入 红队 用例.**

——他 第一次 觉得, **eval 不是 一个 自动 化 的 数字, 是 一个 人机 协 作 的 流水线**. 机器 人 跑, 人 复核, 复核 的 case 喂 回 训练 / prompt 调整. 这 是 一 个 闭环.

"那怎么 确认 eval 集 本身 是 对 的?"阿衍 问.

老周看了他一眼, 那眼神里有一点欣慰.

"——这就是 你 推 门 进来 才 能 问 的 那个 问题. 探头 站 在 门口 的 人 永远 想 不 到. **eval 集 本 身 也 要 治理, 也 要 进化.** 红队 case 攒 起来, 季 度 review 加进 去, 老 case 过 时 退役——**eval 集 是 个 活体, 不 是 一份 静态 文档**.

——这是 生产 化 的 第二 层 难题. 这 是 另一 个 故事."

阿衍 把 那 张 红字 报告 折好, 放进 口袋.

---

## 六

他们离开会议室.

阿衍觉得时间过得飞快——手机一看, **已经是凌晨一点**.

他忽然明白老周为什么今晚一定要带他走一趟.

白天, 他会说"我们用 RAG". 但他根本**不知道 RAG 长什么样, 不知道 rerank 是什么, 不知道 max_steps 能省多少钱, 不知道 eval 集在哪儿放**.

他知道了名词, 但他没走过这个空间.

走过之后, **这些名词才有了重量**.

---

## 七

老周站在走廊尽头.

"最后一层看完了. 跟我上天台."

"天台?"

"对. **我们从地下走到了顶. 是时候看看整栋楼了.**"

阿衍跟上去.

电梯按钮上, 8F 上面还有一个按钮——只是没有灯.

老周按下去.

一丝暖光亮起.
