# 第二层 · 铁匠铺、仓库、走廊

> 这章要讲什么?
> 云上的三件套——计算、存储、网络. 每种资源的"形状"和代价.

---

## 一

二楼是一个环形大厅.

正中间是一座**铁匠铺**, 四台炉子烧着蓝火, 四个铁匠抡着锤子. 第一个铁匠身上穿着厚重的铠甲, 一锤一锤稳稳地打; 第二个轻便许多, 叠罗汉一样一人身上站了三个徒弟; 第三个根本没人——只有一根机械臂按节奏挥锤; 第四个最怪, **不打铁的时候连炉子都是冷的**, 有人丢进来一块铁, 火才腾地一下起来.

"这是什么?"

"计算." 老周说, "从左到右: 裸金属、VM、容器、Serverless. 你选哪一个, 取决于你愿意自己管多少."

阿衍盯着最右边那个冷炉子.

"Serverless 真的能冷启动起来?"

"能. 但你塞一块 '几十 GB 的模型权重' 进去, 它就起不来了." 老周顿了顿, "这事我们八楼再讲."

---

## 二

大厅的左侧是一排**仓库**.

最近一间, 门口堆着**整块的硬盘条**, 一个老师傅坐在凳子上, 给每一块硬盘贴标签, 然后抱进某一台 VM 的机柜里.

"块存储. 一块盘只能一台机器挂."

第二间门口是**铁架子**, 架子上放着文件柜, 一个送货员骑着小车, 同时给三台机器送文件.

"文件存储. 多机共享读写."

第三间就大了. 一整个**自助快递柜**, 从天花板一直延伸到视野尽头, 每个柜子上都有一个扫码口. 有外卖员进来扫码取件, 也有人进来把一个硬盘大小的包裹塞进去.

"对象存储. **给互联网的存储**. 你的图片、备份、数据湖, 都在这种柜子里."

阿衍看着那三间仓库.

**块给一台机、文件给一群机、对象给整个互联网.** 这句话他突然记住了.

---

## 三

大厅的右侧是一条**走廊**, 但走廊的墙不是实墙, 是半透明的.

走廊的入口有一道门, 门口站着一个保安. 保安的工牌上写着 "**Security Group**".

"这条走廊叫 VPC." 老周说, "你自己的私人院子. 外面的人进不来, 除非你开门."

他指了指走廊中间几道**玻璃隔断**.

"子网. 通常分三层: public、private、data. Public 面向互联网, private 放应用, data 放数据库. **层层递进, 层层收紧**."

一个带着互联网帽子的访客过来, 想进走廊. 保安拦住, 看了看他的权限, 放他进了 public 子网, 但不让他进更里面.

"好多人搞不清 SG 和 NACL." 阿衍说, "我一直也没搞清."

老周笑了. "**SG 是每个人的门禁卡, NACL 是每个房间的门牌. SG 管实例, NACL 管子网. SG 是白名单, NACL 有顺序规则.** 你只要记住——SG 是你第一道反应, NACL 是补丁."

---

## 四

阿衍在大厅中间站了一会儿.

三件套摆在眼前, 他忽然意识到: **每一个云上的应用, 不管它叫什么名字, 拆开都只是这三样的不同组合**.

- 一个 Web 服务 = 容器(计算) + Postgres(存储) + VPC(网络)
- 一个 ETL 任务 = Serverless(计算) + S3(存储) + IAM 跨账号(网络)
- 一个视频平台 = VM 集群 + S3 + CDN(edge)

"你的 PPT 第一页其实就应该画这三件套, " 老周说, "然后告诉客户, **我们的所有服务, 都是这三种东西的搭配**. 比 22 页服务图标有用得多."

阿衍心里某个地方, 咔的一声, 对上了.

---

## 五

离开前, 他回头看了一眼.

铁匠铺的火光, 仓库的灯, 走廊的玻璃.

他想, 这些东西, 他以前都"用过". 但这是**第一次, 他看见它们在一栋楼里的位置**.

老周已经走到下一部电梯前了.

"走, 上四楼."

"三楼呢?"

老周没回头. "三楼就是走廊. 网络本来就是一条路, 不是一层楼, 只是图纸上画成一层方便记."

阿衍 还想 问.

老周 在 电梯 里 转过身.

"——我先 跟你 把 这件 事 说 清楚. 这栋 楼 有 两套 编号."

他 在 电梯 镜子 上 哈了 一口气, 用 手指 画:

```
物理楼层      逻辑层 (架构师心里的"层")
─────────    ───────────────────────────
1F  地基     地理 (Region / AZ / Edge)
2F  大厅     三件套 (计算 / 存储 / 网络)
3F  走廊     ── 网络已并入 2F 的走廊
4F  厨房     应用平台
5F  冰库     数据
6F  屏幕墙   横切三柱 (安全 / 观测 / 成本)
7F  空着     ── 横切并入 6F
8F  光井     AI 部署
```

"——**物理楼层 是 给 你 的 脚 走 的**. 一层 一层 上. 但 **逻辑层 不一定 一一对应**: 网络 不 单 占 一层, 它 是 走廊 本身; 横切 三柱 也 不 单 占 一层, 它 们 穿透 整栋楼.

——所以 后面 你 看 我 跳过 三楼 / 七楼, 不是 楼 没了, 是 那 两层 的 内容, 已经 化在 别的 层 里 了."

阿衍 这 才 反应 过来. 他以前 看 的 PPT 都 是 "8 层架构", 假设 每层 都 是 等值 的——其实 **物理 是 8 层, 逻辑 只有 5-6 个 主题, 中间 还有 横切**.

"——记住 这个 区别. 不然 后面 你 会 越走越乱."

电梯 升到 4F.

阿衍跟上去.
