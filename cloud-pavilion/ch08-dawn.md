# 尾声 · 图纸上的下一笔

> 这章要讲什么?
> 从"看过一遍"到"能画出来". Solution Architect 的真正工作.

---

## 一

天台.

风比白天大. 城市在脚下亮着零星的灯, 远处一条河在黑里泛光. 天快亮, 但还没亮, 东边的云有一条很淡的灰白.

老周走到天台边, 背对阿衍.

"你今晚走了八层."

"嗯."

"把你看见的东西, 在心里从头过一遍, 给我画一张图."

阿衍沉默了一会儿. 然后他从口袋拿出笔记本.

---

## 二

他画了一栋楼.

```
┌──────────────────────────────────┐
│  8F  AI 阁楼 (光井穿透下面所有)   │
│    ├ GPU 心跳                     │
│    ├ KV Cache 草稿纸              │
│    ├ RAG 图书馆                   │
│    └ Agent 会议室                 │
├──────────────────────────────────┤
│  7F  [空]  观测并入 6F            │
│  6F  门禁墙 + 监控室 + 账本        │
│  5F  冰库(热/温/冷)              │
│  4F  厨房(无状态厨师 + 队列桶)    │
│  3F  走廊(VPC + 子网)            │
│  2F  铁匠铺 + 仓库                │
│  1F  [并入 2F]                    │
├──────────────────────────────────┤
│  地基:三座城 + 闪电 (Region/AZ/Edge)│
└──────────────────────────────────┘
```

他把笔记本递给老周.

老周看了很久.

"你漏了一件事."

阿衍皱眉.

---

## 三

老周用自己的笔, 在图纸左边补了一行竖着的字:

> **每一层都是取舍.**

"这栋楼, 不是一个标准答案. **它是一千种可能里你选了一种的样子**. 每一层你都在做决定: 用 VM 还是容器? 强一致还是最终一致? OLTP 还是 OLAP? 自托管还是托管 API?"

"Solution Architect 的工作——不是会画这张图, 是**知道这张图里每一格, 换一种画法会发生什么**. 然后对着客户说: '我推荐这样. 原因是 A、B、C. 代价是 D.'"

阿衍把这句话记下来.

**知道每一格换一种画法会发生什么.**

---

## 四

风小了一点.

阿衍问了一个一直想问的问题.

"周哥, 那我下周给客户讲什么?"

老周笑了.

"讲你今晚画的这张图. **不要讲 22 页服务 logo. 讲一栋楼. 讲每一层住谁, 讲光井是怎么穿下来的. 讲你为什么在某一格选了 A 而不是 B.**"

"客户听得懂吗?"

"客户听得懂**空间**, 听不懂**目录**. 你给他看一栋楼, 他一眼就记住. 你给他看一张目录, 他两分钟就忘."

这句话阿衍记了一辈子.

---

## 五

天开始亮.

老周从口袋里拿出一张卡, 递给阿衍.

"电梯的卡. 你以后想进来, 自己进."

"我一个人?"

"对. **这栋楼不是用来参观一次的, 是用来每周走一圈的.** 新技术不停地往上加. 我进来的时候, 还没有 8F. 两年后可能就有 9F——也许是量子计算, 也许是 Neuromorphic. 谁知道."

"那 9F 要盖起来, 也是我们的事?"

老周看着他.

"是你们的事."

---

## 六

阿衍把卡握在手里.

他想起自己今天早上那份 22 页的 PPT. 此刻在他心里, 它已经死了. 明天他会重写——不, 他想, 他会从那张**一栋楼的图**开始写, 写成一份**五页纸的讲稿**.

他低头, 在笔记本的扉页, 写下今晚给自己留下的一句话:

> **做 Solution Architect 不是记更多服务的名字.
> 是在脑子里盖一栋你随时能走进去的楼.**

---

## 七

电梯回到地面.

老周在一楼大堂和他告别.

阿衍走出公司, 路灯已经暗了, 街上空着. 他看了一眼手机——**04:19**.

他想起刚进电梯时, 老周说过的一句话:

> "八楼不是尽头, 是现在的顶."

他抬头看天.

天边是一丝很淡的橙色. 一天要开始了.

他往家走.

脚步比来的时候稳.

---

## 八

他知道, 下一次进这栋楼, 他看见的东西会不一样.

因为**他自己, 已经不一样了.**

——完.
