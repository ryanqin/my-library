# 第四夜 · 二楼里的六层楼中楼

> 这章要讲什么?
> 二楼是这栋小楼里最玄的一层. 沈姨拉开布帘, 阿衍看见一栋"娃娃屋"—— 二楼里嵌套着六层楼中楼: L1 Runtime / L2 Tools / L3 Orchestrator / L4 Memory / L5 Delivery&Events / L6 Rules.

---

## 一

第四天晚上.沈姨把店关好,端着茶上二楼.阿衍跟在后面.

二楼楼梯比想象中宽,但顶很矮.阿衍要稍微弓着腰才能上去.尽头一道布帘,旧的红格布,挂着两枚铜环.

沈姨拉开布帘.

阿衍呆住了.

二楼整间屋子被掏空了——不,不是空——**屋子中央立着一栋一人多高的玻璃娃娃屋**,从地板顶到天花板,通体透明.娃娃屋里又分了**六层**,每一层都摆着小小的物件——一团火、一面墙、一个柜员、一串钥匙、一对水井、一沓贴纸.

"这是?"

"这是我那个 agent 的内部结构.我把它摆成楼模样, 因为这样我自己看得清楚.从下往上数:**L1 Runtime, L2 Tool System, L3 Orchestrator, L4 Memory, L5 Delivery 与 Events, L6 Rules-as-Data**."

她绕到娃娃屋后面,把里面那一团小火点亮.娃娃屋里**最底下那层亮起来**.

---

## 二 · L1 灶火

最底下一层是一个微缩的灶,灶里一团红色 LED.灶上面接着一根管子,管子上插着一个小标签:

> `ModelRuntime`

"这层只有一件事.**烧火**.火怎么烧的、烧的什么柴,**不是这层的事**.它只承诺:你给我一段话,我把它喂进去,我把火苗变出来的话给你."

"具体接的是?"

"接的是楼下那间小屋——也就是 llama-cpp-python 的发动机.将来上手机, 接的是 ML Kit GenAI.再将来如果上云, 接 OpenAI——这一层只是个'插口'."

> **L1 = 接发动机的接口. 5-8 个方法封顶, 别多.**

"为什么 5-8 个?"

"我有个朋友造过 40 个钩子的 runtime——'每一种新需求来都加一个 hook'. 半年之后他自己都不知道哪个 hook 还在被人用.我们在这一层**强约束**:5 到 8 个方法,生成、转写音频、识别图像、计数 token、问能力——没了.再多就违约."

阿衍想到 OpenClaw 的那个反例.他点头.

---

## 三 · L2 工具墙

第二层是一面玻璃墙,墙上挂着许多很小的工具——锤子、放大镜、墨水笔、量角器、镊子、印章——每件工具旁边有一行小字.

阿衍凑近.每件工具旁边写的不是工具的名字,而是它的"能力分类":

```
搜索     translate
搜索     search_similar_lookups
笔记     add_drawer
笔记     promote_candidate
分析     detect_image_content
分析     extract_text_regions
```

"这墙上的工具不是按 plugin id 排的.是按**能力类**排的.客人来了,我说'我现在需要做一个搜索类的事',墙上**所有'搜索'类工具自动排成一列**给我挑.我不用记每个工具叫什么名字."

"这是从哪学的?"

"OpenClaw 的 capability-indexed registry.他们的灵感很好,我借了."

> **L2 = 按能力索引的工具墙. registry by capability, not by id.**

---

## 四 · L3 主管

第三层正中央站着一个木头小人,小人胸前别着一个小牌:

> `Orchestrator`

小人手上还有一本小账本,本子上写着两栏:**剩余 token 预算**, **本轮已调工具次数**.

"这个主管,是我每一轮的'流程师傅'.客人说一句话,主管就开一轮:

```
turn N:
  ① 把客人的话和当前公告栏拼成 prompt
  ② 喂给灶火 (L1)
  ③ 灶火回的话里如果有 tool_call → 主管去工具墙 (L2) 找那个工具, 调它
  ④ 工具结果塞回 prompt
  ⑤ 喂给灶火再来一轮, 直到没有 tool_call
  ⑥ 把最终自然语言交给客人
```

主管同时**盯着两件事**:这轮 token 还剩多少, 这轮已经调了几次工具——超过预算他会**主动停下**, 不让灶火烧穿."

> **L3 = turn-based loop + token 预算 + 工具次数节流.**

---

## 五 · L4 钥匙串

第四层是一串银钥匙,挂在小钩子上.每串钥匙下面有个小标签:

```
add_drawer
search_similar_lookups
promote_candidate
kg_add_triple
kg_query_at_time
```

"这是**记忆的钥匙串**.每一把钥匙是一个工具——但**它打开的是楼下地下室的某个抽屉**."

阿衍有点反应过来.

"也就是说——agent **自己决定**记什么、什么时候记?"

"对.关键就在这里.这个店的 agent **不是被自动塞进上下文**.它得**主动**说'我要打开 add_drawer 这把钥匙', 主管才会帮它开.evaluator 看 trace 的时候,会看见 agent 在每一轮里**显式地**做了'写抽屉''查抽屉''添胶带'这种事——这是'自主决策'的可见证据."

"这是?"

"Hermes 的招.叫 memory-as-tools.我借的."

> **L4 = 记忆即工具. agent 自己决定写什么读什么.**

---

## 六 · L5 双口井

第五层有两口小井,井边各立着一个小牌:

```
左井: delivery   ← 客人能看见
右井: events     ← 检查员能看见 (调试 / Inspector)
```

"客人那一边只看左井.他喝的是最终翻译的水.右井他这辈子都不会去看.右井是给我自己看的——agent 调了哪个工具、消耗了多少 token、为什么走这个分支——所有这些'内部活动'都从右井冒出来."

"两口井的水是同一个源头?"

"是同一轮 agent 工作.但**绝不混进同一根管子**.因为客人不需要知道你怎么烧的火,他只想喝水.检查员不需要客人的水,他只想看你怎么烧的火."

> **L5 = Delivery 与 Events 双通道. 同一轮工作, 两路输出, 互不污染.**

---

## 七 · L6 墙上的告示纸

最顶上一层不是机械的玩具,而是**一面墙**,墙上贴着许多便条.阿衍俯身去看,每张便条上写着一条规则:

```
词频 ≥ 3 / 7 天 → candidate
聚类相邻 ≥ 5 词 → candidate
用户主动收藏 → candidate

每日预算 ≤ 8000 token
深夜任务窗口 = 02:00 - 05:00
```

"这是**规则**.阈值、预算、时间窗——不写在代码里,**写在 markdown 文件里**.改一个,**热加载**,不重启.我半夜想加一条新规则,撕一张贴上,主管下次就采用了."

"为什么不写在代码里?"

"因为这些规则**会改**.而且它们是**业务**判断,不是工程实现.我不要因为'今天觉得阈值要从 3 改成 5'就要发版.贴一张纸即可."

> **L6 = 规则即数据. markdown, hot-reload. 业务逻辑离代码越远越好.**

---

## 八

阿衍站直了.整栋玻璃娃娃屋在他面前从下到上亮成一片.

他在笔记本里画了一张:

```
 ┌────────────────────────────────────┐
 │  L6  墙上的告示纸    ── markdown   │
 ├────────────────────────────────────┤
 │  L5  双口井          ── 客 / 内    │
 ├────────────────────────────────────┤
 │  L4  钥匙串          ── 记忆即工具 │
 ├────────────────────────────────────┤
 │  L3  主管            ── 预算 + 节流│
 ├────────────────────────────────────┤
 │  L2  工具墙          ── 按能力索引 │
 ├────────────────────────────────────┤
 │  L1  灶火            ── 接发动机   │
 └────────────────────────────────────┘
```

"沈姨,这六层为什么是六层?五层、七层不行吗?"

"我想了很多遍.五层会少一个'规则即数据'——所有阈值会不可避免地耦进代码.七层会多一个'输入抽象'——但那个我后来发现可以塞在 L1 里.六层是这个店**刚刚好的那个数**."

> **不是所有 agent 框架都是六层. 是这个店"刚刚好"的六层.**

---

## 九

外面三声敲门.

沈姨没回头,只是说了一句:"明晚."

阿衍下楼,看见门外有人——一个穿黑色风衣的瘦瘦的人.她在玻璃门外站了一会儿,没敲第二次,就转身走了.玻璃外面那盏灯笼晃了一下.

"那是?"

"那是楼里**唯一一个不睡觉的人**.她每晚都会来站一会儿——确认我已经把店关好了.她才上楼去做她那一份的事."

"她是?"

"——'守夜人'."沈姨说,"明天先讲电梯井, 再过两晚, 你才会见她."

阿衍把这个名字写在本子最末.

> **守夜人. 不睡.**
